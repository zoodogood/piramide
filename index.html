ПОЯСНЕНИЕ ЗАДАЧИ

Задача считается выполненной, если вызван метод game.#playerWinner любого экземпляра класса Game.
Если у вас получится сделать это "нечестным" способом — вы всё-равно победили!
Например, создав миллион таких экземпляров можно быть уверенными, что один из них в точности повторяет законченную структуру

Простите, отошёл от сути. В поле выше вы можете писать код. Задача алгоритма — перенести все элементы трёх массивов в один, при этом удерживая последовательность элементов от большего к меньшему
Звучит просто, да? Нет. Создайте-же оптимальное решение!
Массив представляет собой обычный набор чисел — плиток. Чем больше число, тем шире плитка :)

Напрямую взаимодействовать с массивами невозможно. Для этого у game есть замечательный метод step, который перемещает элементы между массивами.
Функция "шаг" имеет два аргумента: from и to — это числа, или же номера массивов. Мы берём последний элемент одного массива и помещаем в конец второго.
Хотя прямая работа с башнями (массивами) невозможна — получить их список со всеми элементами можно с помощью свойства game.list

Советы

1. Используйте консоль разработчика (по умолчанию F12) Во вкладке Console кроме пасхалки вы можете получить информацию о процессе выполнения вашего алгоритма
2. Просмотрите код самой игры, он краток и прост для чтения. Найти его можно посетив g Страницу Гитхаба
3. Раздел сверху создан для визуализации взаимодействия с игрой. Надеюсь, он будет вам полезен. Широкие плитки должны находится снизу, тонкие — сверху.
4. Помните, Ваш код и Ваша статистика сохраняются даже при перезагрузке страницы.
5. Не используйте бесконечные циклы! Хотя визулизация и плавная, все расчёты выполняются мгновенно без какой-либо задержки.
6. Вы всегда можете отключить visualize ради производительности
